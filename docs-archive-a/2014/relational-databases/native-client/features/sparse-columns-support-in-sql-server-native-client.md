 <span data-ttu-id="44acb-167">但是，如果 `queryout` 用于导出并且您引用的稀疏列属于按名称的稀疏列集的成员，则不能执行向类似结构表的直接导入。</span><span class="sxs-lookup"><span data-stu-id="44acb-167">However, if `queryout` is used for export and you reference sparse columns that are members of the sparse column set by name, you cannot perform a direct import into a similarly structured table.</span></span> <span data-ttu-id="44acb-168">这是因为 BCP 使用与\*\* \* 导入操作一致\*\*的元数据，因此无法将 `column_set` 成员列与此元数据匹配。</span><span class="sxs-lookup"><span data-stu-id="44acb-168">This is because BCP uses metadata consistent with a **select \*** operation for the import and is unable to match `column_set` member columns with this metadata.</span></span> <span data-ttu-id="44acb-169">若要单独导入 `column_set` 成员列，您必须对引用所需 `column_set` 列的表定义一个视图，并且必须使用该视图执行导入操作。</span><span class="sxs-lookup"><span data-stu-id="44acb-169">To import `column_set` member columns individually, you must define a view on the table that references the desired `column_set` columns, and you must perform the import operation using the view.</span></span>
