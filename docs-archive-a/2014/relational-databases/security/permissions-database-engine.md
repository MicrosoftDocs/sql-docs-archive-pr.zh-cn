4.  <span data-ttu-id="30a1e-1381">对于该 **安全上下文**，收集为 **权限空间**授予或拒绝的所有权限。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1381">For that **security context**, collect all the permissions that are granted or denied for the **permission space**.</span></span> <span data-ttu-id="30a1e-1382">权限可以明确表述为 GRANT、GRANT WITH GRANT 或 DENY，也可以是隐含或涵盖的权限 GRANT 或 DENY。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1382">The permission can be explicitly stated as a GRANT, GRANT WITH GRANT, or DENY; or the permissions can be an implied or covering permission GRANT or DENY.</span></span> <span data-ttu-id="30a1e-1383">例如，针对架构的 CONTROL 权限隐含对表的 CONTROL，</span><span class="sxs-lookup"><span data-stu-id="30a1e-1383">For example, CONTROL permission on a schema implies CONTROL on a table.</span></span> <span data-ttu-id="30a1e-1384">对表的 CONTROL 则隐含 SELECT。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1384">And CONTROL on a table implies SELECT.</span></span> <span data-ttu-id="30a1e-1385">因此，如果授予了针对架构的 CONTROL 权限，也就授予了对表的 SELECT 权限。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1385">Therefore, if CONTROL on the schema was granted, SELECT on the table is granted.</span></span> <span data-ttu-id="30a1e-1386">如果拒绝了对表的 CONTROL 权限，也就拒绝了对表的 SELECT 权限。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1386">If CONTROL was denied on the table, SELECT on the table is denied.</span></span>  
  
    > [!NOTE]  
    >  <span data-ttu-id="30a1e-1387">列级权限的 GRANT 覆盖对象级的 DENY。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1387">A GRANT of a column-level permission overrides a DENY at the object level.</span></span>  
  
5.  <span data-ttu-id="30a1e-1388">标识 **必需的权限**。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1388">Identify the **required permission**.</span></span>  
  
6.  <span data-ttu-id="30a1e-1389">如果对于 **权限空间** 中的对象，直接或隐式拒绝授予 **安全上下文** 中任何标识 **必需的权限**，则权限检查失败。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1389">Fail the permission check if the **required permission** is directly or implicitly denied to any of the identities in the **security context** for the objects in the **permission space**.</span></span>  
  
7.  <span data-ttu-id="30a1e-1390">如果对于 **权限空间** 中的任何对象，没有拒绝 **必需的权限** 且 **必需的权限** 包含对 **安全上下文**中任何标识直接或隐式的 GRANT 或 GRANT WITH GRANT 权限，则通过权限检查。</span><span class="sxs-lookup"><span data-stu-id="30a1e-1390">Pass the permission check if the **required permission** was not denied and the **required permission** contains a GRANT or a GRANT WITH GRANT permission either directly or implicitly to any of the identities in the **security context** for any object in the **permission space**.</span></span>
